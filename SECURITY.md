# Security Policy

Thank you for helping us keep Dext secure.

## Reporting Security Issues

If you discover a security vulnerability in Dext, please report it by:

1. **Opening a GitHub Security Advisory**: Go to the [Security tab](https://github.com/ztxtxwd/dext/security) and create a new security advisory
2. **Email**: Send details to the maintainer at the email listed in the repository

Please include:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (if available)

## Response Timeline

We will acknowledge receipt of your vulnerability report within 48 hours and will send a more detailed response within 5 business days indicating the next steps in handling your submission.

## Disclosure Policy

- Please give us reasonable time to address the issue before any public disclosure
- We will keep you informed of the progress towards a fix
- Once the vulnerability is fixed, we will coordinate with you on the disclosure timeline

## Attribution

Dext is based on the [MCP Inspector](https://github.com/modelcontextprotocol/inspector) project by Anthropic. For security issues related to the underlying MCP Inspector functionality, you may also want to report them through [Anthropic's HackerOne program](https://hackerone.com/anthropic-vdp).
